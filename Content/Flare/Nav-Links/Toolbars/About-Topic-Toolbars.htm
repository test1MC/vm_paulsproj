About Topic Toolbars

Creating a topic toolbar requires a Topic Toolbar proxy (see About Proxies). You can also use a skin component (for HTML5), a Standard skin (for WebHelp and HTML Help outputs), or a Mobile skin (for WebHelp Mobile output). Choosing the buttons for the toolbar can be done in the proxy, but it can also be done in the skin or skin component (proxy selections have precedence). In addition, you have the option of using a style class to help control the look of the topic toolbar.

Steps for Creating and Editing Topic Toolbars

Steps for Creating and Editing Topic Toolbars

Following are the basic steps involved with creating and editing topic toolbars:

  1. (Optional) Add Skin/Component You can add or edit a skin or skin component, depending on the output and skin type:
    • HTML5 Skins Although topic toolbars can be inserted into most online outputs, only HTML5 uses a Topic Toolbar skin component as part of the process. With HTML5, the settings in the Topic Toolbar skin component have precedence over any settings in the Topic Toolbar Proxy dialog, including the selection of which buttons to be included in the toolbar. See Topic Toolbars in HTML5 Skin Components.

    • Standard and Mobile Skins See Topic Toolbars in WebHelp Outputs and Topic Toolbars in WebHelp Mobile Output.

      See Standard Skin Styles and WebHelp Mobile Skin Styles.

  2. (Optional) Use Style Class You can create and edit a Madcap|topicToolbarProxy style class to control the look of the toolbar. See Using Styles to Determine the Look of Topic Toolbars.

  3. Insert Topic Toolbar Proxy See Inserting and Editing a Topic Toolbar.
