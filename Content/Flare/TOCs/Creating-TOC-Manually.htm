Creating a TOC Manually

You can create a table of contents (TOC) by adding books and links to topics, movies, external files, other TOCs, browse sequences, or other Help systems in any kind of structure you want. Following are steps to create a TOC manually, but you can also create one automatically (see Creating a TOC Automatically).

how to create a toc manually

  1. From the Project Organizer, open your TOC.

  2. Create TOC Entries Using Drag and Drop

    1. Open the Content Explorer or File List window pane.

      (Optional) If you want to choose multiple files at the same time from the Content Explorer, click the Show Files button in the local toolbar. The Content Explorer splits into two halves. On the right half of the Content Explorer, find and select the folder and topic files that you want to include in the TOC.

      (Optional) If you want to choose files from the File List window pane, make sure you float and position it (see Customizing the Workspace) so that you can see both the File List window pane and TOC Editor.

      Make sure you do not select the "Resources" folder in the Content Explorer, which holds your ancillary content files (e.g., images, stylesheets). If you do, that folder and its contents will also be included in the TOC.

    2. Drag topic files (and folders, if applicable) from the Content Explorer or File List window pane to the TOC Editor.
    Create TOC Entries Manually
    • You can manually create different types of TOC entries, such as a book or topic entry. You can also create a topic from the TOC Editor and link it to the TOC at the same time. See Creating TOC Entries.
         Link TOC Entries
    • You can manually link TOC entries to topics, books, and bookmarks. Additionally, you can link an entry to external files or Help systems, browse sequences, other TOCs, other Flare projects, HTML Help systems, and Mimic movies. See Linking TOC Entries.

For print-based output, make sure to also include any topics that you created with TOC, index, or glossary proxies in order to produce those types of generated content in the output.

See Stitching PDFs.

You can add even more TOCs to your project, building them in conjunction with the "master" TOC. This might be a good option if you have an especially large project or if you are working with other authors on the same project.

 What's Next?

After you create a TOC, you can edit various properties for the individual entries within it. See Editing TOC Entries.

You also need to enable TOCs in the skin you want to use for the target. This applies to all types of skins, except for HTML5 Top Navigation skins. See Enabling Tables of Contents in Skins.
