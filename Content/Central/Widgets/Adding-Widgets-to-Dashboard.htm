Adding Widgets to a Dashboard

How to Add a Widget to a Dashboard

    • Home Page Click . The Home page opens.

      OR

    • Project Dashboard Click . The Projects page opens. Click a project name. The Project dashboard opens.
  2. In the local toolbar click . The Select Widgets to Add dialog opens.

  3. Select the widget(s) you want to add to the dashboard.

    For some project-related widgets, if you open the widget on the Home dashboard, the information is initially shown for all projects. But if you open the same type of widget on a Project dashboard, it is initially filtered to show the information for only that project. In addition, there are some widgets that are initially empty, waiting for you to open the widget menu in the upper-right corner and filter it, choosing a project and/or target.

    Build History

    The Build History widget displays a graph of the recent build history for the selected target. To choose a project or target for the widget, start by clicking the widget menu . Then choose Filters, and select either Select Project or Select Target.

    Each bar in the graph represents a different build for the target, with the oldest build on the left and the most recent one on the right. The color of each bar indicates the success of the build:

    • Dark Green Bar The build finished with no errors or warnings (although there may be ignored warnings).
    • Yellow Bar The build was canceled.
    • Red Bar An error prevented the build from finishing.

    The information below the bar graph will change, depending on the bar selected (indicated by an arrow under the bar). You can click any bar in the graph to see its details.

    For each bar, you can view the following details:

    • By The person who queued the build.
    • Duration The length of time it took to complete the build.
    • Warnings/Ignored Warnings/Errors The number of issues found while building the project.
    • Queued The date and time that the build was queued.
    • Started The date and time that the build started.
    • Finished The date and time that the build finished.

    You can also use the buttons at the bottom of the widget:

    • Build Log Opens the log for the selected build so that you can see information about warnings or errors. See Opening the Build Log.
    • View Opens the output for the selected build. See Viewing Output.
    • Download Lets you download the output files for the selected build. See Downloading Builds.

    Data will not display in this widget until you select a project and target, and until you generate the target in Central.

         Builds Activity

    The Builds Activity widget lets you see all of the builds that are in a state of failed, completed, canceled, or queued. The most recent build appears at the top of the widget.

    You can use the widget menu to filter the widget for a single project, as well as for particular states. The 10 most recent builds are displayed by default, but you can change this setting from the widget menu.

    In this widget, you can view the following:

    • Project The name of the project. Click the name to open the Project dashboard. This column is not shown if the widget is filtered to show only one project.
    • Target The output icon (e.g., PDF, HTML5) and name of the target. Click the name of the target to open the Build Management page.
    • Started The date and time the build was started.
    • Status The current build status (e.g., finished, running, terminated).
    • State The state of the build (e.g., building, canceled, failed, complete).
    • Build Progress The progress of the build towards completion. This column displays a colored bar, depending on what happens with the build.
      • Light Green The build is in progress.
      • Dark Green The build finished with no errors (although there may be warnings).
      • Yellow The build was canceled.
      • Red An error prevented the build from finishing.
    • Warnings/Ignored Warnings/Errors The number of issues found while building the project. For more information, click the project name to open the Project dashboard, then open the build log. See Opening the Build Log.

    Data will not display in this widget until you generate a target in Central.

         Live Builds

    The Live Builds widget shows all of the builds that are currently set as “Live.”

    In this widget, you can view or do the following:

    • Target Name The output icon (e.g., PDF, HTML5) and name of the target. Click the name of the target to open the Build Management page.
    • URL The vanity URL for the live build. Click the URL to open the output in a new tab. See Setting Builds to Live or Keep.
    • Download Clicking the icon downloads the output to your computer. Multi-file outputs are downloaded as a ZIP file. See Downloading Builds.
    • Live Date The date the build was set as “Live.”

    Data will not display in this widget until you set a target to "Live."

         Project Bookmarks

    The Project Bookmarks widget allows you to quickly jump to the dashboard for your favorite projects. To add projects to this widget, click the widget menu and choose Select Project.

    Data will not display in this widget until you select a project.

         Project Properties

    The Project Properties widget shows important information about the selected project. If you add this widget to the Home dashboard, you will need to click the widget menu , click Filters, and choose a project.

    In this widget, you can view the following:

    • Created The date the project was created (i.e., added to Central)
    • Last Modified The date the project was last modified (i.e., changes were uploaded to Central, or changes made in Central)
    • Source Project Size The size of the source files of the project (Content and Project Folders)
    • Total Project Size The total size of all the source files and all builds
    • [Project Size Graph] A visual breakdown of how different file types use space in the project. The key below the graph shows how builds, resources, topic files, and project files use the total space.
    • Total Live Builds The number of builds set to "Live" for this project.
    • Total Targets The total number of targets in this project.
    • Total Builds The total number of builds in this project.

    See About Projects.

    If you have this widget on a Project dashboard, you can quickly copy it to the Home dashboard by clicking in the toolbar. See Duplicating Widgets.

    Data will not display in this widget until you select a project.

         Storage and Usage

    The Storage and Usage widget shows important information about the MadCap Central license. This widget is visible to all users, but buying additional seats or space requires the “Purchasing” permission. See Setting User Permissions.

    In this widget, you can view or do the following, depending on your permissions:

    • MadCap Central Expiration Date Shows when the MadCap Central license began and when it will need to be renewed.

      This section displays only if you have an annual subscription (which does not renew automatically). If you are on a monthly subscription (which renews automatically), you will not see this section. To renew your annual subscription, please contact MadCap Software Sales.

    • Users The number of users currently in the system and the number of users allowed. Click Buy Seats to purchase additional user licenses.

    • Storage The amount of storage space used and the amount allowed. Click Buy Space to purchase additional storage space.

    This widget is only available on the Home page.

         Task Calendar

    The Task Calendar widget shows due dates for tasks, based on filter settings that you select from the widget menu .

    In this widget, you can view the following:

    • [Calendar] Corner marks on a date indicate that a task is due on that date. The color of each mark indicates the priority of the task due (red=high, yellow=medium, green=low). Use the arrows on the header bar to navigate between months.
    • Tasks Due In the area below the calendar, you can see a list of tasks and the days they are due, their priority levels, and their titles (which you can click to see details). If you click a date in the Calendar above, this grid will be filtered to show only tasks having that due date. To see all dates in the grid again, click the widget menu and select Clear filters.

    See About Tasks.

    Data will not display in this widget until a task is added on the Tasks page. See Creating and Assigning Tasks.

         Task Summary

    The Task Summary widget shows a doughnut chart that breaks down the percentage of tasks in each milestone and storage area (To Do, In Progress, Complete, Archive, Backlog). To the right of the doughnut chart, you can see a summary with the number of tasks at each priority level (High, Medium, Low).

    See About Tasks.

    Data will not display in this widget until a task is added on the Tasks page. See Creating and Assigning Tasks.
  4. Click Add. The widgets are added to the dashboard.
