How to View the Content of a File

For any file in your project, you can view the markup.-

  1. On the left side of the page, navigate to a file in your project and click on it.

  2. On the right side of the page, click . The XHTML markup for the file is shown.

Example

You just finished publishing an HTML5 target in Central. You take a look at the live build and notice that the header in the first topic is using the wrong style. You could reopen Flare and investigate, but it is quicker to check out the topic in the Repository and see what is wrong.

You navigate to the file in the Repository and select Content. The file's markup appears on the right side of the page.

You are familiar with XHTML, so you quickly scan the markup. Sure enough, the header was switched to a Heading 2 instead of a Heading 1.

This is an easy fix, so you use the Message Center in Central to ask another writer on your team to make the update and push their changes to Central.

Once the changes are made, you double-check the Repository, and see that the incorrect header is now correctly set to a Heading 1.

 How to View the History of a Single file

For any file or folder in your project, you can view each commit to Central after changes are made to the file locally. (A commit means to record your Flare files to the local repository.)

  1. On the left side of the page, navigate to a file in your project and click on it.

  2. On the right side of the page, click .

    A list of the commits is shown below, with the most recent commit shown first. From this list, you can see who made the change. Also, if the individual added a comment when making the push to Central, it is shown; otherwise, generic text is shown indicating that commit was made.

  3. For any commit, you can click the automated number associated with it.

    This changes the view, showing the files that are part of the commit on the left and information about the commit on the right.

  4. To return to the folder/file view, click in the toolbar.

Example

You are documenting some new software and want to see the last time a specific topic was edited. You open the Repository in Central and navigate to that topic, then select History. A list of the topic's commits appears on the right side of the page.

You can see that you made changes to the topic last week, but the topic hasn't been changed at all this week. However, there have been several changes to the software in the last couple of days, and you need to edit this topic reflect those changes.

Now that you know that your topic needs to be updated, you can open it in Flare and make the necessary changes, then upload the changes to Central.

 How to View the History of All Files in the project

In addition to viewing the history for a file or folder, you can also view the history of all files in the project. You can filter the history to view only your commits, or all of the commits in the project.

  1. In the toolbar, click .

    A list of all commits is shown below, with the most recent commit shown first. From this list, you can see who made the change. Also, if the individual added a comment when making the push to Central, it is shown; otherwise, generic text is shown indicating that commit was made.

  2. (Optional) At the top of the page, you can click if you only want to see the changes you’ve made. You can click if you want to see changes from all users.

  3. For any commit, you can click the automated number associated with it.

    This changes the view, showing the file in question on the left and information about it on the right.

  4. To return to the folder/file view, click in the toolbar.
